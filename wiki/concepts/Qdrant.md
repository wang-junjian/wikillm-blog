# Qdrant

Qdrant 是一个生产就绪的开源向量相似性搜索引擎，使用 [[Rust]] 语言编写，以高性能、可扩展性和丰富的过滤能力著称。作为 [[RAG-检索增强生成]] 架构中的核心基础设施，Qdrant 负责存储和检索高维向量，将语义搜索能力引入各类 AI 应用。Qdrant 提供了 REST API 和 gRPC 接口，支持在向量上附加结构化 payload 进行混合过滤，是构建生产级语义搜索系统的首选方案之一。

与 [[FAISS]] 等纯库方案不同，Qdrant 是一个完整的服务化产品——自带持久化存储、分布式部署、访问控制、监控告警等企业级特性。与 [[Chroma]] 等嵌入式方案相比，Qdrant 在性能和可扩展性上更具优势，适合大规模生产环境。Qdrant 团队还开发了 [[FastEmbed]] 嵌入库，与 Qdrant 深度集成，提供端到端的嵌入生成与检索能力。

## 核心概念

### 向量搜索与相似度计算

Qdrant 的核心能力是高效的向量相似性搜索。它将文本、图像、音频等数据通过嵌入模型映射为高维向量，在向量空间中通过距离度量（[[余弦相似度]]、欧氏距离、点积）找到最相似的项。Qdrant 采用 [[HNSW]]（Hierarchical Navigable Small World）近似最近邻索引算法，在保持高召回率的同时实现亚秒级查询响应，即使面对数百万甚至数十亿条向量也能保持高性能。

### Payload 过滤

Qdrant 的独特优势在于对结构化 payload 的原生支持。每个向量可以附加任意 JSON 格式的 payload（元数据），搜索时可以在向量相似度检索的基础上叠加结构化过滤条件：

- **范围过滤**：价格区间、日期范围、评分阈值
- **精确匹配**：类别标签、状态值、用户 ID
- **全文过滤**：字符串包含、正则匹配
- **地理过滤**：经纬度距离、区域范围
- **组合条件**：多个条件的 AND/OR/NOT 逻辑组合

这种"向量搜索 + 结构化过滤"的混合搜索能力（Hybrid Search）是生产环境中的核心需求——例如搜索"价格低于 100 元且评分 4 星以上的蓝牙耳机"。

### 分布式与持久化

Qdrant 从设计之初就考虑了生产环境的需求：

- **持久化存储**：向量数据持久化到磁盘，服务重启后无需重建索引
- **分片与复制**：支持水平分片（Sharding）和数据复制，实现高可用和负载均衡
- **快照备份**：支持集群级快照，便于数据迁移和灾难恢复
- **访问控制**：基于 API Key 的访问控制，支持只读/读写权限分离
- **监控集成**：暴露 Prometheus 指标，便于与现有监控体系集成

### 多语言客户端

Qdrant 提供了多种编程语言的官方客户端 SDK：

- **Python**：`qdrant-client`，最完善的客户端，支持同步/异步操作
- **TypeScript/JavaScript**：`qdrant-js`，适用于 Node.js 和浏览器环境
- **Rust**：`qdrant-rs`，原生 Rust 客户端
- **Go**：`qdrant-go`，Go 语言客户端

其中 Python 客户端还提供了 `qdrant-client[fastembed]` 扩展，可直接集成 [[FastEmbed]] 嵌入模型，实现端到端的嵌入生成与检索。

### 稀疏向量与混合检索

Qdrant 支持稀疏向量（Sparse Vector）存储和检索，可与稠密向量实现混合检索。稀疏向量通常来自 [[BM25]]、[[SPLADE]] 等稀疏编码方案，在关键词精确匹配上表现优异。Qdrant 的混合检索允许在同一次查询中同时使用稠密向量和稀疏向量，通过 Reciprocal Rank Fusion（RRF）或可学习的加权策略融合结果，兼顾语义理解和关键词匹配。

## 应用场景

- **RAG 知识库**：存储文档嵌入，为 [[RAG-检索增强生成]] 系统提供语义检索能力
- **推荐系统**：基于用户/物品嵌入的相似性推荐
- **图像搜索**：以图搜图、相似商品查找
- **异常检测**：通过向量距离识别异常样本
- **问答系统**：FAQ 语义匹配、智能客服知识检索
- **药物分子搜索**：基于分子嵌入的相似化合物发现
