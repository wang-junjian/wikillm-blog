# 模型合并

模型合并（Model Merging）是一类将多个预训练或微调后的模型参数直接融合，创建一个具有多模型能力的新模型的技术。与传统的 [[集成学习]]（Ensembling）需要同时运行多个模型不同，模型合并通过参数空间的操作将多个模型压缩为单个模型，在不增加推理成本的前提下整合多源能力。这一领域在 2023-2024 年间迅速发展，成为 [[开源 LLM]] 生态中的重要工具。

模型合并的核心动机是：在开源社区中，存在大量针对不同任务微调的模型（如代码模型、数学模型、对话模型、多语言模型），如果能够将这些模型的能力融合到一个模型中，就可以避免为每个任务维护独立模型的成本。模型合并提供了一种低成本、高效能的能力整合方案——无需额外训练，仅通过参数操作即可实现。

## 核心概念

### Task Vector 理论

[[Task Vector]] 是模型合并的理论基石，由 Meta AI 团队提出：

- **定义**：Task Vector τ 是微调后模型权重 θ_ft 与预训练模型权重 θ_pre 的差值：τ = θ_ft - θ_pre
- **直觉**：Task Vector 编码了微调过程中学到的"任务特异性知识"
- **合并原理**：将一个模型的 Task Vector 加到另一个模型上，可以实现能力迁移
- **算术操作**：Task Vector 支持加法（能力叠加）、减法（能力移除）、缩放（能力强度调整）

Task Vector 的发现揭示了模型参数空间的线性结构——微调产生的变化在参数空间中具有方向性和可加性，这为模型合并提供了理论基础。

### TIES-Merging

[[TIES]]（TrIm, Elect, and Merge）是一种解决参数干扰的模型合并方法：

1. **修剪（Trim）**：保留 Task Vector 中幅度最大的参数，将接近零的参数置零（去除噪声和冗余）
2. **选举（Elect）**：对多个模型在同一参数位置上的符号（正/负）进行投票，取多数符号
3. **合并（Merge）**：将修剪和选举后的 Task Vector 加到基础模型上

TIES 解决了多模型合并时的参数冲突问题——不同微调模型在同一参数方向上可能产生相反的变化，直接平均会相互抵消。通过符号选举机制，TIES 保留了多数模型认可的变化方向。

### DARE（Drop And REscale）

[[DARE]] 是一种简单而有效的模型合并技术：

- **随机丢弃**：将 Task Vector 中的大部分参数随机置零（通常丢弃 90%-99%）
- **缩放补偿**：将保留的参数按比例放大（乘以 1/drop_rate），保持总体变化幅度
- **直觉**：微调产生的参数变化中，只有少量是关键的，大量是噪声。随机丢弃相当于一种正则化

DARE 常与 TIES 结合使用（DARE-TIES），在保留关键变化的同时去除噪声，在多个基准上取得了优异的合并效果。

### SLERP（Spherical Linear Interpolation）

[[SLERP]] 是一种在球面空间中进行线性插值的合并方法：

- **原理**：将模型权重视为高维空间中的向量，沿球面进行插值
- **优势**：相比线性插值（LERP），SLERP 更好地保持了权重向量的范数特性
- **应用**：特别适合合并两个模型，产生介于两者之间的能力

SLERP 是最早的模型合并方法之一，虽然简单但在许多场景下效果良好。

### MergeKit 工具

[[MergeKit]] 是开源社区中最流行的模型合并工具库，由 Goddard 开发：

- **支持的合并方法**：TIES、DARE、SLERP、Linear、Task Arithmetic、Passthrough 等
- **YAML 配置**：通过简单的 YAML 配置文件定义合并策略
- **LazyMerges**：支持分块加载大模型，降低内存需求
- **Kitchen Sink**：支持同时合并多个模型，每种模型赋予不同权重

MergeKit 极大降低了模型合并的技术门槛，使普通开发者也能尝试模型融合实验。

### Model Soups

[[Model Soups]] 是 Google Research 提出的模型合并方法：

- **均匀汤（Uniform Soup）**：对同一任务微调的多个模型进行简单平均
- **贪心汤（Greedy Soup）**：逐步添加能提升验证集性能的模型子集
- **发现**：简单平均多个微调模型的效果往往优于选择最佳单个模型

Model Soups 表明，不同随机种子微调产生的模型在错误上具有多样性，平均可以互补错误、提升泛化能力。

## 应用场景

- **多任务能力融合**：将代码、数学、对话等专项模型合并为通用模型
- **多语言能力整合**：合并不同语言微调的模型，实现多语言支持
- **模型集成替代**：在不增加推理成本的前提下获得集成学习的收益
- **能力组合创新**：探索不同模型能力组合产生的新特性
- **模型压缩**：将多个专家模型合并为单一模型，降低部署成本
- **开源社区协作**：不同团队训练的模型通过合并快速组合能力
