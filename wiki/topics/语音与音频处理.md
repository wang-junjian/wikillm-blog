# 语音与音频处理

> 领域：大模型与算法研究

## 概述

语音与音频处理是人工智能中连接声学信号与自然语言的关键领域，涵盖语音识别（ASR）、语音合成（TTS）、语音翻译、说话人分离和音频事件检测等核心任务。本主题聚合从 Whisper 到 FunASR、从云端 API 到端侧 C++ 部署的完整工程实践，覆盖离线转写、实时流式识别以及多模态语音对话系统。

语音处理生态呈现明显的分层格局：以 Whisper 为代表的多语言通用模型、以 FunASR 为代表的中文优化全链路工具包、以 whisper.cpp 为代表的端侧推理方案，以及以 SeamlessM4T 为代表的多模态翻译模型。选型时需在识别精度、推理延迟、部署成本和语言覆盖之间做出权衡。2025-2026 年，语音识别从"转文字"走向"理解并行动"——ASR 不再孤立存在，而是作为多模态交互系统的输入层，与 LLM、运动控制和视觉感知深度耦合。

## 关键知识点

1. **Whisper 通用基座** — OpenAI 多语言语音识别（tiny 到 large-v3）、云端 API（25MB 限制）vs 本地部署、长音频静音检测分割（PyDub detect_silence）
2. **whisper.cpp 端侧推理** — C/C++ 原生实现、NEON + Metal / CoreML / MLX 加速路径对比、FP32 到 Q5_1 量化选择
3. **WhisperLiveKit 实时服务** — FastAPI + WebSockets 架构、说话人识别（Diarization）、WSS 流式传输、Jetson Thor 边缘部署（torchaudio 编译约束）
4. **SimulStreaming** — AlignAtt 同步策略、同声传译（音频未结束即可输出译文）、`--comp_unaware` 计算预算控制
5. **FunASR 全链路** — AutoModel 多模型协调引擎、VAD 分段 → ASR 识别 → 标点恢复 → 说话人分离流水线、2pass 模式（实时 + 高精度修正）
6. **SenseVoice 多维度理解** — ASR + LID + SER + AED 四合一、富文本识别（情感/事件标签）、非自回归端到端框架（10 秒音频 70ms 推理）
7. **PaddleSpeech** — 飞桨语音开源库、中文 TTS 流式合成、numpy 版本兼容性
8. **SeamlessM4T** — Meta 近百种语言翻译（ASR/S2ST/S2TT/T2ST/T2TT）、Apple Silicon MPS 限制（语音输入任务需 CPU/CUDA）
9. **语音对话系统** — Reachy Mini 分层实时对话机器人（LLM 后端 + 头部运动/编舞/情感/摄像头工具 + 60Hz 控制环路）
10. **音频工程实践** — FFmpeg 格式统一（16kHz 单声道 16-bit PCM）、容器化部署（FunASR CPU/GPU 双镜像）、audio2sub 字幕工具（VTT/SRT 批量输出）
