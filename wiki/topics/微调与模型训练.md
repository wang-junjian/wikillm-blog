# 微调与模型训练

> 领域：LLM 与生成式 AI
> 状态：完整

## 概述

微调（Fine-tuning）是在预训练 LLM 基础上，通过特定领域数据的继续训练，使模型适应专项任务的技术路径。与提示工程的"零参数更新"范式不同，微调直接修改模型权重，能够更深刻地改变模型的行为模式与知识边界。当提示工程无法满足需求——需要改变模型的内在行为风格、注入大量领域知识、或实现特定任务的性能提升时，微调成为必然选择。

本主题覆盖微调的完整技术栈：从 LoRA/QLoRA/P-Tuning v2 等参数高效微调方法，到 LLaMA Factory/SWIFT/MLX 等微调框架，再到命名实体识别（NER）/Text2SQL 等实践案例，以及 Easy Dataset 数据制备、ONNX Simplifier 模型优化、Hugging Face NLP 课程、LLMOps、合成数据、分布式训练、Sky-T1 推理模型训练等前沿方向。

## 关键知识点

1. **[[LoRA]]（Low-Rank Adaptation）**——在冻结的原始权重旁注入低秩矩阵，仅训练模型总参数的 0.01%–5%，在消费级硬件上微调大模型成为可能
2. **[[QLoRA]]（Quantized LoRA）**——在 LoRA 基础上引入 4-bit 量化，融合后模型更小（4G vs 13G）、推理速度提升约 6 倍
3. **[[P-Tuning v2]]**——通过可训练的前缀编码器引导模型输出，仅微调约 0.1% 的参数
4. **[[LLaMA Factory]]**——集成多种微调方法的统一框架，通过 Web UI 极大降低技术门槛
5. **[[SWIFT]]**——ModelScope 生态下的全流程微调框架，支持近 200 种 LLM 和多模态大模型
6. **[[MLX]]**——Apple 为 Apple Silicon 设计的机器学习框架，统一内存架构支持本地 LoRA 微调
7. **[[Easy Dataset]]**——面向大模型微调的数据集构建工具，利用 LLM 智能分割文本、批量生成问题
8. **[[Sky-T1-32B-Preview]]**——以不到 450 美元训练成本达到 o1-preview 推理水平的开源推理模型
9. **[[合成数据]]**——通过 LLM 生成的训练数据，Cosmopedia 包含超 3000 万个文件、250 亿个词元
10. **[[DeepSpeed]]**——微软的分布式训练优化库，支持 ZeRO 系列并行策略，使有限硬件下训练大模型成为可能
