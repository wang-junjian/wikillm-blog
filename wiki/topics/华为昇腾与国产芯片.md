# 华为昇腾与国产芯片

> 领域：AI 硬件与芯片加速

## 概述

本主题系统梳理华为昇腾（Ascend）NPU 的大模型部署全栈实践，涵盖硬件架构、驱动固件安装、MindIE 推理引擎、LLM 部署、集群化管理及性能调优。同时涉及海光 DCU 推理压测、盘古大模型关键技术解读，以及 GlusterFS、GPUStack、Docker Swarm 等分布式部署方案。

华为昇腾软件栈自底向上分为驱动层（CANN）、运行时层（Ascend Runtime）、推理引擎层（MindIE/vLLM-ascend）和应用层。Atlas 800I A2 是面向大模型推理的旗舰设备——鲲鹏 920 CPU + 昇腾 910B4 NPU 异构架构，单机 8 张 NPU 卡（每卡 32GB HBM），总显存 256GB。Atlas 900 则面向超大规模训练集群。

## 关键知识点

1. **硬件平台** — Atlas 800I A2（鲲鹏 920 + 昇腾 910B4 × 8、256GB HBM、1024GB 内存、openEuler 22.03）、Atlas 900 AI 集群（384 NPU + InfiniBand）
2. **软件栈架构** — CANN（驱动/固件/MCU）→ Ascend Runtime → MindIE（Service/LLM/Turbo/Torch）/ vLLM-ascend → OpenAI 兼容 API
3. **部署流程** — ① LVM 逻辑卷（XFS）→ ②③ 驱动/固件安装（首次 vs 覆盖顺序差异）→ ④ MCU 升级 → ⑤ Docker 安装 → ⑥ 镜像导入 → ⑦ 模型下载 → ⑧ 服务部署
4. **MindIE 推理** — 单实例部署（设备文件映射、config.json 配置）、多实例部署（端口分配、NPU 绑定、envsubst 模板化）、Docker Compose 部署
5. **vLLM-ascend** — 张量并行约束（注意力头数整除）、最大序列长度、内存碎片优化（PYTORCH_NPU_ALLOC_CONF）、DeepSeek-V3 部署注意事项
6. **性能基准** — TTFT/TPOT/ITL/吞吐量指标、W8A8 量化 vs Float16 对比、DeepSeek-R1-7B vs Qwen2.5-7B 性能接近
7. **集群管理** — GPUStack（多厂商 GPU/NPU 兼容、自动故障恢复）、Docker Swarm 分布式部署（overlay 网络 + firewalld 冲突）
8. **共享存储** — GlusterFS 无元数据服务器设计、分布式复制卷、千兆网卡瓶颈（建议 25GbE 升级）
9. **海光 DCU** — K100_AI 64GB 显存、HIP_VISIBLE_DEVICES 环境变量、Qwen2.5-7B/72B 压测、注意力头数整除约束
10. **算力切分** — vNPU 技术（npu-smi 切分模板）、多容器共享物理卡、算力隔离与资源共享权衡
