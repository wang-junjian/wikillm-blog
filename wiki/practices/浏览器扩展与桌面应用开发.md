# 浏览器扩展与桌面应用开发

浏览器扩展与桌面应用开发是工程师将工具链延伸到用户界面的关键能力。本页面以 [[Visual Studio Code]] 扩展开发为核心，聚合 [[VS Code API]] 参考、扩展示例和开发实践。

## Visual Studio Code 扩展开发

[[Visual Studio Code]]（VS Code）是最流行的代码编辑器之一，其扩展生态覆盖语言支持、主题、调试器和工具集成。通过开发扩展，可将自定义工作流直接嵌入编辑器。

### 开发资源

- **[[VS Code API]]**：提供编辑器操作、窗口管理、文件系统访问等完整接口
- **[[VS Code Extension Samples]]**：官方示例仓库，涵盖各类扩展场景

### 技术栈

VS Code 扩展使用 [[JavaScript]] 或 [[TypeScript]] 开发，运行在独立的扩展宿主进程中。核心概念包括：

- **Activation Events**：控制扩展何时激活（如打开特定文件、执行命令）
- **Commands**：注册可通过命令面板调用的操作
- **Language Server Protocol（LSP）**：实现代码补全、跳转、诊断等语言功能
- **Webview**：在编辑器内渲染自定义 UI

### 开发流程

1. 使用 `yo code` 脚手架生成项目模板
2. 实现 `extension.ts` 中的 `activate` 注册命令和贡献点
3. 通过 `F5` 启动扩展开发宿主进行调试
4. 使用 `vsce` 打包并发布到 [[VS Code Marketplace]]

## 浏览器扩展

浏览器扩展基于 Web 技术（[[HTML]]、[[CSS]]、[[JavaScript]]），通过浏览器提供的 API 增强浏览能力。主流浏览器（[[Chrome]]、[[Firefox]]、[[Edge]]）支持 [[WebExtensions API]]。

### 核心概念

- **Manifest**：`manifest.json` 声明扩展元数据、权限和资源
- **Background Scripts**：后台脚本处理事件和状态管理
- **Content Scripts**：注入网页上下文执行 DOM 操作
- **Popup / Options Page**：扩展的用户界面

## 桌面应用

桌面应用开发有多种技术路径：

- **[[Electron]]**：使用 Web 技术构建跨平台桌面应用（如 VS Code、Slack）
- **[[Tauri]]**：基于 Rust 的轻量级替代方案，体积更小
- **[[Flutter]]**：Google 的 UI 框架，支持桌面、移动和 Web
- **原生开发**：macOS 使用 Swift/Objective-C，Windows 使用 C#/C++

选型时需在开发效率、应用体积、原生能力和跨平台需求之间做出[[权衡（Trade-off）]]。
