# 引用链接与科技评论

> 本聚合页收录科技领域核心人物（Karpathy、Paul Graham、Georgi Gerganov、Simon Willison、Andrew Quinn）的推文与脚注、开发者工具生态（Excalidraw、MinerU、SkVM、DSpark、Stitch、PhotoRoom、GraphvizOnline）、教育工具（高中英语3500词）、Reachy Mini 机器人实战，以及 Karpathy 的 CLAUDE 编码准则。

## 1. Andrej Karpathy：AI 时代的软件工程

### 1.1 Software 3.0 范式

[[Andrej Karpathy]] 在 2025 年提出**软件三范式**框架：

- **Software 1.0**：传统手工编码（Python、C++）
- **Software 2.0**：基于神经网络权重训练（梯度下降调参）
- **Software 3.0**：通过自然语言提示编程大语言模型

LLM 被类比为**新型操作系统**，同时具备三种特性：

| 特性 | 类比 | 说明 |
|------|------|------|
| 公用事业 | 电网 | CAPEX 训练 + OPEX 服务 + 按量计费 |
| 晶圆厂 | 台积电 | 巨额资本支出 + 深度技术树研发 |
| 操作系统 | macOS/Windows | 复杂软件生态系统 + 切换摩擦 |

LLM OS 架构：LLM 扮演 CPU 角色，上下文窗口相当于 RAM，经典软件工具（计算器、Python、终端）作为外围应用，提示空间类比内核/用户空间。

### 1.2 Claws：AI 技术栈的新层级

Karpathy 在 2026 年 2 月提出 **Claw** 概念——类似 OpenClaw 的个人 AI 智能体系统，通常在本地硬件上运行，通过消息协议通信。

**核心类比**：LLM Agent 是 LLM 之上的新层级 → Claw 是 LLM Agent 之上的新层级。Claw 将编排、调度、上下文管理、工具调用和持久性提升到新高度。

生态变体：
- **NanoClaw**：核心引擎约 4000 行代码，可控、可审计、灵活，默认容器化运行
- **nanobot / zeroclaw / ironclaw / picocaw**：其他轻量级变体

Karpathy 的洞察：「配置方案不是通过配置文件，而是通过技能实现」——输入 `/add-telegram` 指令 AI Agent 修改代码集成 Telegram，避免配置混乱。

### 1.3 LLM 知识库实践指南

Karpathy 在 2026 年 4 月分享 **LLM Knowledge Bases** 方法论：
- 将源文件（文章、论文、代码、图片）归档到 `raw/`
- LLM 逐步"编译"生成知识库（按目录组织的 .md 文件）
- 知识库包含摘要、反向链接、概念分类、互链文章
- 使用 [[Obsidian]] 作为前端 IDE
- 无需复杂 RAG——LLM 自动维护索引和文档摘要
- 输出归档回知识库（Markdown、Marp 幻灯片、Matplotlib 图表）

### 1.4 CLAUDE.md 编码准则

Karpathy 发布的 `CLAUDE.md` 文件用于改善 [[Claude Code]] 行为，核心原则：

1. **编码前先思考**：明确假设、呈现权衡、不确定的就提问
2. **简单至上**：最少代码解决问题，不添加推测性内容
3. **外科手术式修改**：只碰必须改的，只清理自己弄乱的
4. **目标驱动执行**：定义成功标准，循环直到验证通过
5. **不要假设包已安装**：检查依赖、主动引入
6. **追踪 token 成本**：关注上下文窗口使用

## 2. 科技界核心人物观点

### 2.1 Paul Graham：AI 代笔的欺骗感

[[Paul Graham]] 在 2026 年 5 月推文：

> "我现在收到的很多创始人邮件都写成了一种强硬的新闻体风格。我知道它们是 AI 写的，因为以前从来没有创始人这样写过。一旦你意识到某件事是 AI 写的，就很难不去忽略它。"

核心观点：
- 用 AI 代笔邮件是欺骗，降低对作者的评价
- 任何青少年都能用 AI 写东西——这不令人印象深刻
- 投资人需要警惕 AI 生成的标准化沟通

### 2.2 Georgi Gerganov：本地模型的编程价值

[[Georgi Gerganov]]（[[GGML]]/llama.cpp 创始人）在 [[Hacker News]] 评论：

> "我可以 100% 证实，Qwen3.6-27B 在处理编程任务时是一款非常出色的本地模型。在过去一个半月里，我几乎每天都在使用它。"

使用场景：
- 在 M2 Ultra 和 RTX 5090 上本地运行
- 处理 ggml-org 中简单乏味的日常任务
- 使用轻量级 pi agent 环境 + 系统提示词定制风格
- 本地模型在代码辅助编程中的实用价值已非常显著

### 2.3 Andrew Quinn：重新发明轮子的价值

[[Andrew Quinn]] 在关于 SQLite 替换为 FST 的文章脚注：

> "你**必须**去重新发明几个轮子，才能触及人类对'造轮子'这件事的认知边界。"

核心论点：
- 盲目学习无法替代亲手实践
- 每个重新发明的轮子和针对性问题，会把人推向真正的技术前沿
- 速度之快，是盲目死板学习无法比拟的
- 数学/CS 领域可能需要重新发明 20-30 个轮子

## 3. Simon Willison：极客博客与知识管理

### 3.1 博客设计哲学

[[Simon Willison]]（[[Django]] 联合创始人、[[Datasette]] 作者）的博客是个人知识库的典范：

**六种内容类型**：Entry（长文）、Blogmark（链接）、Quotation（引文）、Note（笔记）、Beat（外部活动）、Chapter（指南章节）。

**融合机制**：
- 首页统一时间流——所有类型按天分组混排
- 标签是唯一组织维度
- 极简排版——系统字体、黑蓝灰三色、22 年风格一致
- SQL `UNION ALL` 合并六种类型为统一时间流

### 3.2 技术关注重点

- AI 辅助编程与 Agent 架构
- [[MCP]] 协议与权限控制
- 本地大模型与端侧运行
- Datasette 生态（ACL 权限、Agent 集成）

### 3.3 博客重构启示

对 wangjunjian.com 的借鉴：
- 引入 `links`、`quotes`、`notes` 三个新集合
- 首页改为统一时间流
- 标签系统优化——合并过于细化的标签
- 利用 [[Astro]] Content Collections 实现多类型内容管理

## 4. 开发者工具生态

### 4.1 Excalidraw：手绘风格虚拟白板

[[Excalidraw]] 是一款开源虚拟白板工具，主打手绘风格图表绘制：

- **核心能力**：无限画布 + 手绘质感线条 + 实时多人协作 + 端到端加密
- **技术栈**：TypeScript + React，MIT 许可证，126k+ Stars
- **生态集成**：Obsidian 插件、VS Code 扩展、Notion、Google Cloud
- **导出格式**：PNG、SVG、.excalidraw JSON、只读分享链接
- **适用场景**：系统架构图、流程图、UML、线框图、技术配图

### 4.2 MinerU：高精度文档解析引擎

[[MinerU]]（OpenDataLab 开源）为 LLM/RAG/Agent 提供结构化数据：

- **输入**：PDF、DOCX、PPTX、XLSX、图片、网页
- **输出**：Markdown、JSON（阅读顺序）、多模态 Markdown
- **核心能力**：去除页眉页脚、多栏识别、表格/图片/公式提取、109 语言 OCR
- **双引擎**：`pipeline`（快且省资源）/ `hybrid-engine`（精度更高）
- **性能**：OmniDocBench v1.6 整体 86.47（pipeline）/ 95.39（hybrid high）
- **芯片适配**：兼容 Ascend、Cambricon 等 10+ 国产 AI 芯片
- **生态集成**：MCP Server、LangChain、LlamaIndex、RAGFlow、Dify

### 4.3 SkVM：面向 LLM 技能的编译系统

[[SkVM]] 首次为 LLM Agent 的 Skill 构建跨模型、跨平台的编译与运行时系统：

**核心类比**：Skill = 自然语言程序，LLM = 异构处理器，SkVM = 编译器 + 运行时。

**量化发现**（118,000+ Skills）：
- 15% 的任务使用 Skill 后性能下降
- 87% 的任务至少一个模型无改善
- Token 开销最高暴增 451%

**三大创新**：
1. 26 个原始能力画像 → 差距分析 → 补偿/替换
2. 代码固化：75% Skill 包含可固化代码片段 → 延迟降低 19-50×
3. 并发提取：DLP/ILP/TLP 并行加速最高 3.2×

**评估结果**：8 个 LLM × 3 个 Harness，任务完成率 +15.3%、Token 消耗 -40%。

### 4.4 DSpark：半自回归投机解码

[[DSpark]]（北京大学 + DeepSeek 联合开发）结合高吞吐并行生成与自适应负载感知验证：

**核心架构**：
- 半自回归生成：并行骨干 + 轻量串行头（Markov 头 / RNN 头）
- 置信度调度验证：置信度预测头 + 序列温度缩放校准
- 硬件感知前缀调度器：全局吞吐量最大化

**性能**：
- 离线：相比 Eagle3 接受长度 +26.7%~30.9%
- 生产环境：DeepSeek-V4 线上单并发速度 +60%~85%
- 开源：DeepSpec 仓库集成 Eagle3、DFlash、DSpark

### 4.5 Google Stitch：AI 原生 UI 设计工具

[[Stitch]] 将文字、草图、截图、语音指令转化为高保真界面与前端代码：

- **三步提示词公式**：[想法] + [主题] + [内容]
- **迭代方式**：每次只改动一点，用 UI/UX 词汇精准指令
- **交付**：导出 HTML/图片包，HTML 可转为 React/Vue/Flutter/SwiftUI
- **定位**：打通「灵感→设计→开发」完整工作流

### 4.6 PhotoRoom：AI 驱动抠图平台

[[PhotoRoom]] 以「一键 AI 抠图 + 智能背景替换」为核心：

- **即时背景移除**：处理发丝、毛绒、透明物体等复杂边缘
- **AI 背景生成**：基于扩散模型文字生成匹配背景
- **电商集成**：Shopify/Amazon/Etsy 一键导出规范尺寸
- **商业数据**：估值超 5 亿美元、累计下载 1.5 亿次、融资 6400 万美元

### 4.7 GraphvizOnline：浏览器端图可视化

[[GraphvizOnline]] 是纯客户端的在线 Graphviz 编辑器：

- DOT 代码即时渲染、多格式导出（SVG/PNG/DOT）
- Viz.js 引擎 + d3-graphviz 动画 + Ace-editor 语法高亮
- URL 分享、Gist 加载、演示模式嵌入
- 零数据外传，所有渲染在浏览器本地完成
