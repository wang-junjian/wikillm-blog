# Q&A 与知识沉淀详细指南

## Q&A 执行清单（严格按顺序执行）

- [ ] **步骤 1**：确认当前任务是 Q&A 场景（用户在提问，而非要求编译文档）
- [ ] **步骤 2**：运行 `python3 tools/wikillm.py search <问题关键词>` 检索 wiki
- [ ] **步骤 3**：精读命中的 wiki 页面，进行跨文档推理
- [ ] **步骤 4**：生成回答，包含：
  - 对问题的直接解答
  - "参考文档清单"（使用 Wikilink 格式）
- [ ] **步骤 5**：判断是否需要归档：
  - 是否具有通用研究价值？
  - 是否可能被其他人再次查询？
  - 如果是 → 继续步骤 6；如果否 → 结束
- [ ] **步骤 6**：自动归档：
  - 将 Q&A 整理为一篇新的 markdown 文章
  - 存入 `wiki/queries/YYYY/` 目录（按年份分子目录）
  - 文件名使用 kebab-case，如 `How-to-Do-Something.md`
  - 在 `INDEX.md` 的"Q&A 归档"部分创建入口

## 检索优先级

回答前必须按以下顺序检索：

1. **`python3 tools/wikillm.py search <query>`**：优先使用 CLI 搜索工具，它会基于文件名、标题、标签和正文加权返回最相关页面
2. **主题枢纽页**：查看 `wiki/topics/` 下是否有相关主题页
3. **Glossary**：查看 `wiki/Glossary.md` 中是否有核心术语
4. **grep 全局搜索**：当搜索工具未命中时使用

## Q&A 归档文档的元数据格式

```yaml
---
title: "问题标题（用中文）"
source: "WikiLLM Q&A"
date: YYYY-MM-DD
tags:
  - "Q&A"
  - "其他标签"
question: |
  在这里记录原始问题
---
```

## Q&A 场景的子判断

- **简单查询**：可以直接用现有知识回答，无需创建新文档 → 仅回答，不归档
- **复杂查询**：答案具有通用研究价值，可能被其他人再次查询 → 回答 + 归档到 `wiki/queries/YYYY/`

## 详细流程

**当用户针对 Wiki 发起复杂查询时**：

1. **检索**：先运行 `python3 tools/wikillm.py search <关键词>`，获取候选页面
2. **精读**：阅读候选 wiki 页面和相关的主题枢纽页
3. **回答**：不仅要解决当前问题，还需提供"参考文档清单"
4. **自动归档 (Filing)**：若该次 Q&A 具有通用研究价值，LLM 需自动将其整理为一篇新的文章，存入 `wiki/queries/YYYY/`，并在 `INDEX.md` 中创建入口

## 回答模板

```markdown
## 直接回答

[简明扼要地回答用户问题]

## 详细解释

[展开说明，引用 wiki 页面]

## 参考文档

- [[Concept-Name|概念名]] — 说明该文档与问题的关系
- [[Practice-Name|实践页]] — 说明该文档与问题的关系
- [[Topic-Name|主题枢纽页]] — 了解该主题的完整脉络
```

## 主题枢纽页中的 Q&A 入口

每个 `wiki/topics/*.md` 主题枢纽页可以包含"相关问答"小节，把高价值的 queries 页面链接回来：

```markdown
## 相关问答

- [[Q-What-Is-RAG|什么是 RAG？]]
- [[Q-How-To-Deploy-vLLM|如何部署 vLLM？]]
```

## 注意事项

- 简单问题可以直接回答，不必为了归档而归档
- 归档时必须保留原始问题，方便日后检索
- 回答中涉及的每个核心概念都必须有 Wikilink
- 如果搜索工具未命中但 grep 发现了相关内容，应反思是否需要把该内容整理成更独立的 wiki 页面
